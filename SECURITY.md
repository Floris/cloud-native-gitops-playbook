# Security Policy

This repository contains guidance and templates, not production credentials.

## Reporting

If you find insecure guidance, open a private security report where possible or raise an issue that clearly explains:

- what is unsafe
- why it is unsafe
- what safer alternative should be used

## Scope

Security issues in this repository typically include:

- unsafe Kubernetes defaults
- overly permissive RBAC guidance
- insecure supply chain recommendations
- examples that normalize secret sprawl or cluster-admin usage
